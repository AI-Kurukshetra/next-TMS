# NextGen TMS Design

Use this skill when changing layout, styling, or interaction design in this repository.

## Design Intent
Build a logistics dashboard that feels operational, clear, and purposeful. Avoid template-looking admin screens and avoid decorative noise that hurts scanning speed.

## Priorities
1. Fast scanning of shipment state
2. Clear assignment and exception visibility
3. Strong hierarchy in tables and forms
4. Reliable mobile and desktop behavior

## Rules
- Prefer high-contrast status communication.
- Keep forms compact, labeled, and action-oriented.
- Every important table should surface status and route/location context.
- Use whitespace and grouping to separate booking, assignment, and tracking concerns.
- Avoid purple-heavy defaults, low-contrast text, and vague empty states.
- Do not add motion unless it helps orientation.
